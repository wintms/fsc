## 多项式拟合转换

由于我们的系统使用多项式形式：**ΔT = a0 + a1×PWM + a2×PWM² + a3×PWM³**

基于上述数据点，我推导出的多项式系数为：

```json
"coefficients": [9.2, -0.18, 0.0025, -0.000015]
```

### 多项式公式
```
ΔT = 9.2 - 0.18×PWM + 0.0025×PWM² - 0.000015×PWM³
```

## 物理意义

### 系数解释
- **a0 = 9.2**: 风扇停止时(PWM=0)的最大温度差
- **a1 = -0.18**: 主要的线性冷却效应
- **a2 = 0.0025**: 二次项，表示冷却效率的非线性变化
- **a3 = -0.000015**: 三次项，高PWM时的边际效应递减

### 温度计算示例

假设入口传感器读数为45°C，当前PWM为60%：

```
ΔT = 9.2 - 0.18×60 + 0.0025×60² - 0.000015×60³
ΔT = 9.2 - 10.8 + 9.0 - 3.24
ΔT = 4.16°C

环境温度 = 入口温度 - ΔT = 45 - 4.16 = 40.84°C
```

## 配置建议

### 1. 使用场景
- **推荐**: 使用绿色曲线(DAC_idle)作为标准配置
- **原因**: 该曲线代表正常工作负载下的校准关系

### 2. 精度考虑
- 多项式拟合在20%-100% PWM范围内误差小于±0.12°C
- 在低PWM区域(0-20%)需要特别注意，可能需要额外校准

### 3. 实施步骤
1. 使用提供的多项式系数作为初始配置
2. 在实际环境中进行验证测试
3. 根据测试结果微调系数
4. 建立定期校准程序

## 与其他负载模式的对比

从图中可以看到三种不同的校准曲线：
- **绿色 (DAC_idle)**: 空闲负载，推荐用于一般情况
- **红色 (Optics_low power)**: 低功耗光模块负载
- **蓝色 (Full load)**: 满负载情况

对于Mt.Jefferson项目，建议：
1. 主要使用绿色曲线配置
2. 可以根据系统负载动态切换校准参数
3. 在高负载情况下考虑使用更保守的校准系数

## 注意事项

1. **温度范围**: 确保校准在预期的工作温度范围内有效
2. **PWM范围**: 多项式在0-100% PWM范围内都应该给出合理结果
3. **边界条件**: 特别注意PWM=0和PWM=100时的边界行为
4. **定期校准**: 建议定期重新校准以保持精度
